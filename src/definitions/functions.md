# Functions

## Generalized Types and Parameter Environments

A _generalized type_ $\defn{gty}{\gty}$ is either a type $\ty$ or a region $r$

A _param env_ $\defn{paramenv}{\paramenv}$ is a list of _constraints_; where a constraint takes the form:

- $\region : \region'$ ($\region$ outlives $\region'$)
- $\ty: \region$ (All regions in $\ty$ outlive $r$)
- $\ty: \text{Tr}$ ($\ty$ implements $\text{Tr}$)

### Outlives Judgments

The _base outlives_ relation $\paramenv \vdash_0 \gty : \region$ holds iff it
can be derived from the following rules:

1. **Direct**: $(\gty : \region) \in \paramenv$
2. **Reflexivity**: $\paramenv \vdash_0 \region : \region$
3. **Transitivity**: $\paramenv \vdash_0 \gty : \region$ and $\paramenv \vdash_0 \region : \region'$ implies $\paramenv \vdash_0 \gty : \region'$

That is, the base outlives relation is the transitive closure of the
region-outlives and type-outlives facts in $\paramenv$.

There also exists an _outlives_ relation $\paramenv \vdash \gty : \region$ such
that $\paramenv \vdash_0 \gty : \region$ implies $\paramenv \vdash \gty :
\region$. This relation is computed by Rust's type system and additionally
accounts for outlives constraints implied by trait membership constraints in
$\paramenv$.

## Function Signatures

A _function signature_ is a pair $\langle\fargtys,~\fresty\rangle$.

A _defined function signature_ $f$ is a tuple $\langle \textit{defid}, ~\fargtys,~\fresty, \paramenv \rangle$.

An _instantiation_ $\defn{funcinst}{\hat{f}}$ of $f$ is the tuple $\langle f,
\overline{\gty}\rangle$; where $\overline{\gty}$ is a list of _early-bound parameters_.

The _identity instantiation_ $f_I$ of $f$ is obtained by applying the _identity substitution_ $I_\gty$.  _Defined function calls_ are applied to _instantiations_ of a function.

The _generalized lifetime projections_ $\defn{glprojs}{\glprojs(\funcinst{f})}$ of a function instantiation $\hat{f}$ is defined as the set:

$\{\lproj{\text{arg}~i}{\glft}~|~ i \leqslant |\fargtys|, \glft \in \glfts(\fargty{i})|\} \cup $
$\{\lproj{\text{result}}{\glft}~|~\glft \in \glfts(\fresty)\}$

## Function Calls

A _function call target_ $\tilde{f}$ is either an instantiation $\funcinst{f}$ or a closure / function pointer $ct$.

A _function call_ $\defn{fncall}{\fncall}$ takes the form $p =
\tilde{f}(\overline{op})~\text{at}~l$, where $p$ is a MIR place, and
$\overline{op}$ is a sequence of MIR operands.

$\paramenv(\fncall)$ is the parameter environment of the function _with respect to the call site_.

The _lifetime projections_ $RP(\fncall)$ of a function call is the union of the lifetime projections in $p$ and the lifetime projections in $\overline{op}$.

A function call $\fncall$ is valid iff it satisfies the _unique region property_: each region in the lifetime projections of $\fncall$ is unique.

<div class="warning">
We assume that function calls generated by directly extracting the result place
and operands from a MIR body are valid.
We note that converting the places to PCG places (which use the type derived from their local), does not necessarily maintain the validity of a function call.
</div>
